# 建议

- 使用stm32CubeMX直接生成rtthread项目的方式，缺陷很多，不建议，下面使用官方的源码直接根据bsp构建项目

## 构建工具

- pip install scons

## 源码

- https://github.com/RT-Thread/rt-thread

## toolchain

- [下载地址](https://developer.arm.com/tools-and-software/open-source-software/developer-tools/gnu-toolchain/gnu-rm/downloads)
- 修改项目目录的`rtconfig.py` , EXEC_PATH，指向toolchian的bin目录即可
- 测试发现，用系统自带的也可以，不一定需要这个toolchain，下面这行配置也是可以的

```bash
EXEC_PATH   = r'/usr/bin/'
```

## 编译

```bash
scons -c; scons;
```

## 如需修改配置

```bash
scons --menuconfig;
```

## 生成独立源码包

- 不依赖当前目录，可以复制到任何地方编译

```bash
scons --dist
```

## env工具配置

- 目前只支持python2

```bash
sudo apt-get install make build-essential libssl-dev zlib1g-dev libbz2-dev \
libreadline-dev libsqlite3-dev wget curl llvm libncurses5-dev libncursesw5-dev \
xz-utils tk-dev libffi-dev liblzma-dev python-openssl git

pyenv install 2.7.16

cd ~/.env/
pyenv local 2.7.16

code ~/.profile
export PATH=~/.env/tools/scripts/:$PATH

source ~/.profile
```

## 断点调试

- 见vscode_template目录配置，无论是jlink还是daplink，openocd全系列支持
- stm32f103c8比较特殊，需要在stm32f1xx_hal_msp.c里面修改HAL_MspInit函数，删除__HAL_AFIO_REMAP_SWJ_DISABLE函数的调用；如果忘记删除也没事，burn或者erase之前，按住rst按钮，再进行burn或者erase，再松开rst按钮即可。
