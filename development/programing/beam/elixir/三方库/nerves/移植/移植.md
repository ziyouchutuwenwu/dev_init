# 移植

## 说明

基于 buildroot，但是不需要先编译 buildroot

参考 [结构](https://github.com/nerves-project/nerves_system_bbb)

```sh
$HOME/.nerves 是全局 nerves 数据，比如预编译的 toolchain
.nerves 是某个 bsp 的构建缓存
```

## 步骤

### 生成

```sh
# 下载缓存
export BR2_DL_DIR=$HOME/downloads/port/downloaded_cache
export MIX_ENV=prod
```

mix.exs 里面，注释掉，否则 deps 每次都会去下载编译好的 bsp

```elixir
artifact_sites: [
  {:github_releases, "#{@github_organization}/#{@app}"}
],
```

清理

```sh
rm -rf .nerves
rm -rf ./deps
```

依赖，会自动下载 toolchain

```sh
mix deps.get
```

```sh
# 目前执行 clean, 它会自动下载源码，应该是 bug
# 名字是 mix.exs 里面，app 字段
mix nerves.clean nerves_system_bbb

# 进入 buildroot 的源码目录，可以在里面手动 make
mix nerves.system.shell

# 触发buildroot构建
mix nerves.artifact
```

### 引用

在 .nerves/artifacts 下找到 tar.gz 包

targets 字段对应 MIX_TARGET

```elixir
def deps do
  [
    {:xxx, path: "xxxxxx", runtime: false, targets: :bbb}
  ]
end
```

或者

```sh
mv .nerves/artifacts/xxx.tar.gz ~/.nerves/artifacts/
```

```elixir
def deps do
  [
    {:xxx, runtime: false, targets: :bbb}
  ]
end
```

或者

```sh
export NERVES_ARTIFACTS_DIR=xxxxxxxxxxx
```

```elixir
def deps do
  [
    {:xxx, runtime: false, targets: :bbb}
  ]
end
```
