# 固件升级

## 说明

需要用 [nh](https://github.com/nerves-hub/nerves_hub_cli) 来推送固件

## 步骤

### 生成密钥

保存好

```sh
# 生成 fwup-key.pub 和 fwup-key.priv
fwup -g
```

### 固件处理

配置

```elixir
config :nerves_hub_link, client: FirmwareUpgrader
```

更新逻辑，根据情况，看是否要更新

```elixir
defmodule FirmwareUpgrader do
  use NervesHubLink.Client
  require Logger

  #  :apply 直接更新
  #  :ignore 此次忽略
  #  {:reschedule, timeout_in_milliseconds} xx 毫秒以后重试

  @impl NervesHubLink.Client
  def update_available(data) do
    case :rand.uniform(3) do
      1 ->
        Logger.info("接受更新 data #{inspect(data)}")
        :apply

      2 ->
        Logger.info("延迟 10 秒 data #{inspect(data)}")
        {:reschedule, 10_000}

      _ ->
        Logger.info("忽略更新 data #{inspect(data)}")
        :ignore
    end
  end
end
```

改版本号

```sh
mix.exs 修改 @version
```

```sh
export MIX_TARGET=x86_64
export MIX_ENV=prod
rm -rf _build
mix firmware
```

固件签名

```sh
fwup -S -s fwup-key.priv \
  -i _build/x86_64_prod/nerves/images/nerves_demo.fw \
  -o _build/x86_64_prod/nerves/images/nerves_demo_signed.fw
```

初始固件烧录并运行

```sh
# 本地烧录签过名的固件
fwup -d disk.img _build/x86_64_prod/nerves/images/nerves_demo_signed.fw

qemu-system-x86_64 \
  -enable-kvm \
  -m 1024 \
  -drive file=disk.img,if=virtio,format=raw \
  -netdev bridge,id=net0,br=virbr0 \
  -device virtio-net-pci,netdev=net0 \
  -nographic \
  -serial mon:stdio
```

### 发布准备

nh 的配置目录 `$HOME/.nerves-hub`，如果 nh 有问题，可以参考

登录

```sh
nh user auth
```

本地 key 导入

```sh
nh key import prod_key fwup-key.pub fwup-key.priv
```

### 发布固件

固件部分，修改版本号，重新签名，但是 **不需要** 在设备上烧录

上传新版固件即可，可以看到 uuid

```sh
nh firmware publish --product nerves_demo _build/x86_64_prod/nerves/images/nerves_demo_signed.fw
```

查看，拿到 uuid

```sh
nh firmware list
```

创建 deployment

```sh
# conditions 和 tags 都不是必须的
# uuid 是必须的
nh deployment create
```

### 固件更新

切换状态，满足条件的设备会自动升级

```sh
nh deployment update 0.2.0 state on
```

查看 deployment

```sh
nh deployment list --product nerves_demo
```
