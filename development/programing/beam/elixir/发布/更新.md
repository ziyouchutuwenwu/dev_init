# 更新

## 说明

依赖 heart

如下变量，都需要 unset

```sh
RELEASE_VSN
Release 版本号，如 0.1.0。脚本用这个值确定 $REL_VSN_DIR="$RELEASE_ROOT/releases/
$RELEASE_VSN"——从哪里加载启动脚本、sys.config、vm.args。这是最关键的一个，错了整个启动
路径全错。

RELEASE_SYS_CONFIG
Erlang 的 -config 参数指向的文件路径（不带 .config 后缀），即 sys.config 的位置。这个
文件定义应用的运行时配置。泄漏了就会用旧版本的配置启动新版本。

RELEASE_VM_ARGS
Erlang 的 -args_file 参数指向的文件路径，即 vm.args 的位置。这个文件传给 Erlang VM 参
数（比如 +Q 65536 这种）。泄漏了就用旧版本的 VM 参数。

RELEASE_REMOTE_VM_ARGS
remote.vm.args 的路径，和上面一样，但只在 bin/demo remote 或 bin/demo rpc 时使用。泄漏
后连接到新版本会用旧版本的 remote VM 参数。

RELEASE_COOKIE
Erlang 分布 cookie。读写自 $RELEASE_ROOT/releases/COOKIE。泄漏后新 daemon 用的是旧
cookie，但 bin/demo remote 从新 release 读 cookie，两边不一致，就连不上。

RELEASE_TMP
临时目录路径，默认 $RELEASE_ROOT/tmp。daemon 模式用 --pipe-to 把 pipe 和日志写在这里。
泄漏后新 daemon 的日志和 pipe 会写在旧版本的 tmp 目录里，不会导致启动失败，但旧目录被
清理时管道就断了。
```

## 步骤

推荐的目录结构

编译以后，rel 目录下，对应具体版本号

```sh
/opt
├── demo-0.1.0
│   ├── bin
│   ├── erts-16.3.1
│   ├── lib
│   └── releases
└── demo-0.2.0
    ├── bin
    ├── erts-16.3.1
    ├── lib
    └── releases
```

代码

```elixir
defmodule Release.Upgrader do
  def to(version) do
    with {:ok, cmd} <- build_cmd(version) do
      :heart.set_cmd(cmd)
      System.halt(1)
    end
  end

  def build_cmd(version) do
    root = System.get_env("RELEASE_ROOT")
    name = System.get_env("RELEASE_NAME")
    if root && name do
      target = Path.join([Path.dirname(root), "#{name}-#{version}", "bin", name])
      if File.exists?(target) do
        {:ok, upgrade_cmd(target)}
      else
        {:error, {:target_not_found, target}}
      end
    else
      {:error, :missing_env}
    end
  end

  # env -u 清除环境变量，让新 release 自己设置
  # 如果希望跨版本，就不要清
  defp upgrade_cmd(target) do
    ~c"env \
        -u RELEASE_COOKIE \
        -u RELEASE_VSN \
        -u RELEASE_SYS_CONFIG \
        -u RELEASE_VM_ARGS \
        -u RELEASE_REMOTE_VM_ARGS \
        -u RELEASE_TMP \
        #{target} daemon"
  end
end
```

测试

```sh
/opt/demo-0.1.0/bin/demo daemon
/opt/demo-0.1.0/bin/demo remote

# 执行
Release.Upgrader.to "0.2.0"

# 查看
/opt/demo-0.2.0/bin/demo remote
```
