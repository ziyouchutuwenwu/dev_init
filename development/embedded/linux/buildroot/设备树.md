# 设备树

## 说明

linux 内核自带设备树的路径，build 以后才能看到

```sh
# 被 include 的 dts 在这里
output/build/linux-*/arch/<ARCH>/boot/dts/

# 编译结果
output/images/
```

## 配置

### 准备

```sh
make menuconfig
```

```sh
# 启用设备树
Kernel  --->
  [*] Build a Device Tree Blob (DTB)
  (board/demo_vendor/demo_board/dts) Out-of-tree Device Tree Source overlay directories
```

推荐方案 2

### 方案 1

准备目录

```sh
board/demo_vendor
└── demo_board
    ├── demo_board.defconfig
    └── dts
        └── demo_board.dts
```

demo_board.dts

```sh
# 虚拟机，只能 dump
qemu-system-aarch64 -M virt -cpu cortex-a53 -machine dumpdtb=/tmp/virt.dtb
dtc -I dtb -O dts /tmp/virt.dtb > ./demo_board.dts
```

真实板子，手写

```dts
/dts-v1/;

#include "arm/rtsm_ve-aemv8a.dts"

/ {
    model = "demo board";
    compatible = "demo_vendor,demo_board";
};
```

```sh
make linux-rebuild
```

### 方案 2

dts 目录下建立一个厂商的子目录，就和 内核树 `output/build/linux-*/arch/<ARCH>/boot/dts/` 的组织形式一致

```sh
board/demo_vendor
└── demo_board
    ├── demo_board.defconfig
    └── dts
        └── demo_vendor
            └── demo_board.dts
```

demo_board.dts

```dts
/dts-v1/;

// 注意这里的路径
#include "../arm/vexpress-v2p-ca9.dts"

/ {
    model = "demo board";
    compatible = "demo_vendor,demo_board";
};
```

```sh
make linux-rebuild
```
