# 分离

## 说明

配置文件全部独立到外部目录

读取 `.config`， 把非默认值的选项写到你指定的文件

uboot，kernel, busybox 分别读取自己目录下的 .config

最佳实践：开始就分离配置

## 分类

### 准备

独立配置存放的目录

```sh
mkdir -p board1
```

external.desc

```sh
name: XXX
```

```sh
touch Config.in external.mk
```

设置环境变量，后面不要加 /

```sh
export BR2_EXTERNAL=$(pwd)/../demo_board
```

### uboot

```sh
make menuconfig
```

设置路径

```sh
Bootloaders  --->
  U-boot  --->
    U-Boot configuration --->
      (X) Using a custom board (def)config file
    ($(BR2_EXTERNAL_XXX_PATH)/config/uboot_config) Configuration file path
```

配置文件

```sh
# 自己画的板，先 touch

# 现成的板
# qemu_arm64_defconfig
# 配置文件如果找不到，可以 make uboot-extract
```

### kernel

```sh
make menuconfig
```

设置路径

```sh
Kernel  --->
  Kernel configuration ()  --->
    (X) Using a custom (def)config file
  ($(BR2_EXTERNAL_XXX_PATH)/config/linux_config) Configuration file path
```

配置文件

```sh
# 自己画的板，先 touch

# 现成的板
# board/qemu/aarch64-virt/linux.config
# 配置文件如果找不到，可以 make linux-extract
```

### rootfs

```sh
make menuconfig
```

设置路径

```sh
Target packages  --->
  ($(BR2_EXTERNAL_XXX_PATH)/config/busybox.fragment) Additional BusyBox configuration fragment files
```

配置文件

```sh
# touch 即可
# busybox.fragment 是增加覆盖默认的配置
```

### 整体配置

保存

先做各种配置，最后执行

```sh
make BR2_DEFCONFIG=$BR2_EXTERNAL/config/config savedefconfig
```

还原

也是最后做，否则 .config 会被其他配置覆盖

```sh
make BR2_DEFCONFIG=$BR2_EXTERNAL/config/config defconfig
```
