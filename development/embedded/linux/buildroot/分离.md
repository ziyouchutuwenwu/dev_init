# 分离

## 说明

配置文件全部独立到外部目录

读取 `.config`， 把非默认值的选项写到你指定的文件

uboot，kernel, busybox 分别读取自己目录下的 .config

最佳实践：开始就分离配置

## 分类

### 目录准备

独立配置存放的目录

```sh
mkdir -p board1
```

external.desc

```sh
name: AAA
```

```sh
touch Config.in external.mk
```

设置环境变量，后面不要加 /

```sh
export BR2_EXTERNAL=$(pwd)/../board1
```

### uboot

```sh
make menuconfig
```

设置路径

```sh
Bootloaders  --->
  U-boot  --->
    U-Boot configuration --->
      (X) Using a custom board (def)config file
    ($(BR2_EXTERNAL_AAA_PATH)/uboot_defconfig) Configuration file path
```

配置文件如果找不到，可以执行 **make uboot-extract**

```sh
# 自己画的板，先 touch

# 现成的板
# qemu_arm64_defconfig
```

详细配置

```sh
make uboot-menuconfig
```

保存同步到独立的文件，必须先跑上面的 xxx-menuconfig

```sh
make uboot-update-defconfig
```

### kernel

```sh
make menuconfig
```

设置路径

```sh
Kernel  --->
  Kernel configuration ()  --->
    (X) Using a custom (def)config file
  ($(BR2_EXTERNAL_AAA_PATH)/linux_defconfig) Configuration file path
```

配置文件如果找不到，可以执行 **make linux-extract**

```sh
# 自己画的板，先 touch

# 现成的板
# board/qemu/aarch64-virt/linux.config
```

详细配置

```sh
make linux-menuconfig
```

保存同步到独立的文件，必须先跑上面的 xxx-menuconfig

```sh
make linux-update-defconfig
```

### rootfs

```sh
make menuconfig
```

设置路径

```sh
Target packages  --->
  ($(BR2_EXTERNAL_AAA_PATH)/busybox.config) BusyBox configuration file to use?
```

占位

```sh
# 自己画的板，先 touch
# 默认的复制 busybox.config
```

详细配置

```sh
make busybox-menuconfig
```

保存同步到独立的文件

```sh
make busybox-update-config
```

### 板级配置

保存

先做各种配置，最后执行

```sh
make BR2_DEFCONFIG=$BR2_EXTERNAL/demo_board.defconfig savedefconfig
```

还原

也是最后做，否则 .config 会被其他配置覆盖

```sh
make BR2_DEFCONFIG=$BR2_EXTERNAL/demo_board/demo_board.defconfig defconfig
```
