# 配置保存

## 说明

读取 `.config`， 把非默认值的选项写到你指定的文件

uboot，kernel, busybox 分别读取自己目录下的 .config

最佳实践：开始就分离配置

## 分类

### uboot 配置

```sh
make menuconfig
```

设置路径

```sh
Bootloaders  --->
  U-boot  --->
    U-Boot configuration --->
      (X) Using a custom board (def)config file
    (board/demo_vendor/demo_board/uboot_defconfig) Configuration file path
```

文件占位

```sh
# 自己画的板
touch board/demo_vendor/demo_board/uboot_defconfig

# 复制 output/build/uboot-xxx/configs/qemu_arm64_defconfig
cp -rf output/build/uboot-xxx/configs/xxx_defconfig board/demo_vendor/demo_board/uboot_defconfig
```

详细配置

```sh
make uboot-menuconfig
```

保存

```sh
make uboot-update-defconfig
```

### 内核配置

```sh
make menuconfig
```

设置路径

```sh
Kernel  --->
  Kernel configuration ()  ---> (X) Using a custom (def)config file
  (board/demo_vendor/demo_board/linux_defconfig) Configuration file path
```

文件占位

```sh
# 自己画的板
touch board/demo_vendor/demo_board/linux_defconfig

# 复制配置文件过去
cp -rf board/qemu/aarch64-virt/linux.config board/demo_vendor/demo_board/linux_defconfig
```

详细配置

```sh
make linux-menuconfig
```

保存

```sh
make linux-update-defconfig
```

### busybox 配置

```sh
make menuconfig
```

设置路径

```sh
Target packages  --->
  (board/demo_vendor/demo_board/busybox.config) BusyBox configuration file to use?
```

占位

```sh
# 自己画的板
touch board/demo_vendor/demo_board/busybox.config

# 复制
cp -rf package/busybox/busybox.config board/demo_vendor/demo_board/busybox.config
```

详细配置

```sh
make busybox-menuconfig
```

保存

```sh
make busybox-update-config
```

### 板级配置

保存

先做各种配置，最后执行

```sh
make BR2_DEFCONFIG=board/demo_vendor/demo_board/demo_board.defconfig savedefconfig
```

还原

也是最后做，否则 .config 会被其他配置覆盖

```sh
make BR2_DEFCONFIG=board/demo_vendor/demo_board/demo_board.defconfig defconfig
```
