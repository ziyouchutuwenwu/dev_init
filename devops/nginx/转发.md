# 转发

## 说明

一般用于 ssl 转发 到后端

## 步骤

生成后端证书

```sh
openssl genrsa -out cert.key 2048
openssl req -x509 -new -key cert.key \
  -out cert.crt \
  -days 365 \
  -subj "/C=CN/ST=北京/L=北京/O=组织/CN=aaa.com"
```

```python
import http.server
import ssl

httpd = http.server.HTTPServer(('0.0.0.0', 8443), http.server.SimpleHTTPRequestHandler)
context = ssl.SSLContext(ssl.PROTOCOL_TLS_SERVER)
context.load_cert_chain("../cert/cert.crt", "../cert/cert.key")
httpd.socket = context.wrap_socket(httpd.socket, server_side=True)
httpd.serve_forever()
```

nginx 配置

```nginx
# 后端配置 ssl 证书
# http 字段内配置 ssl 证书，不能和这个混用，必须换端口
stream {
    ssl_preread on;

    server {
        listen 443;
        proxy_pass $name;
    }

    map $ssl_preread_server_name $name {
        aaa.com    ssl_aaa;
        bbb.com    ssl_bbb;
    }

    upstream ssl_aaa {
        server 127.0.0.1:8443;
        server 127.0.0.1:8444;
    }

    upstream ssl_bbb {
        server 127.0.0.1:8445;
        server 127.0.0.1:8446;
    }
}
```
